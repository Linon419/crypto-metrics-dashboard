# 加密货币指标监控 Telegram 机器人

这个 Telegram 机器人可以监控加密货币的指标变化，并在满足特定条件时发送通知。

## 主要功能

### 📊 查询功能
- **币种查询** (`/check <币种>`): 查询单个币种的详细信息
- **市场概览** (`/latest`): 获取最新的市场数据概览
- **收藏管理**: 添加、删除和查看收藏的币种

### 🔔 自动通知功能
- **优质进场期机会提醒**: 全市场高质量进场期币种机会推送
- **爆破指数由负转正提醒**: 全市场爆破指数从负数转为正数的币种
- **爆破指数跌破200提醒**: 收藏币种爆破指数破200时通知
- **退场期重点提醒**: 收藏币种进入退场期时发送提醒

### 🛠️ 管理功能
- 用户订阅管理
- 管理员统计和手动检查功能

## 安装和部署

### 1. 环境要求
- Node.js 16+ 
- NPM

### 2. 创建 Telegram 机器人
1. 在 Telegram 中找到 [@BotFather](https://t.me/BotFather)
2. 发送 `/newbot` 创建新机器人
3. 按提示设置机器人名称和用户名
4. 获取 Bot Token（格式类似：`123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`）

### 3. 安装依赖
```bash
cd telegram-bot
npm install
```

### 4. 配置环境变量
复制配置文件模板：
```bash
cp .env.example .env
```

编辑 `.env` 文件，填入必要信息：
```env
# Telegram Bot Token（必填）
TELEGRAM_BOT_TOKEN=your_bot_token_here

# API 配置（必填）
API_BASE_URL=http://localhost:3001/api

# 管理员 Chat ID（可选，逗号分隔多个）
ADMIN_CHAT_IDS=123456789,987654321
```

**注意**: 不再需要预先配置API认证，用户将在使用时直接输入自己的Dashboard账户凭据。

### 5. 获取管理员 Chat ID
为了使用管理员功能，你需要获取自己的 Chat ID：

1. 启动机器人后，给机器人发送任意消息
2. 查看控制台日志，会显示发送者的 Chat ID
3. 将 Chat ID 添加到 `.env` 文件的 `ADMIN_CHAT_IDS` 中

### 6. 启动机器人
```bash
npm start
```

或者使用开发模式（自动重启）：
```bash
npm run dev
```

## 使用说明

### 首次使用（认证设置）
1. **启动机器人**: 发送 `/start` 查看欢迎信息
2. **连接账户**: 发送 `/auth` 开始认证流程
3. **输入凭据**: 按提示输入您的 Crypto Dashboard 用户名和密码
4. **开始使用**: 认证成功后即可使用所有功能

### 基础命令
- `/start` - 启动机器人并查看欢迎信息
- `/help` - 查看帮助文档
- `/auth` - 设置或查看账户认证状态
- `/check BTC` - 查询 BTC 的详细信息
- `/latest` - 获取市场概览

### 认证管理
- `/auth` - 查看认证状态或开始认证
- `/reauth` - 重新认证（如果凭据过期）
- `/logout` - 注销登录，清除本地凭据

### 收藏管理
- `/favorite ETH` - 将 ETH 添加到收藏
- `/unfavorite BTC` - 从收藏中移除 BTC  
- `/myfavorites` - 查看收藏列表

### 通知订阅
- `/subscribe` - 开启自动通知
- `/unsubscribe` - 关闭自动通知
- `/status` - 查看当前状态

### 管理员命令
- `/admin_check` - 手动触发一次检查
- `/admin_stats` - 查看机器人统计信息

## 通知规则

### 全市场重要时间通知
**高质量进场期机会通知**
- **触发条件**: 任何币种处于高质量进场期前3天
- **通知内容**: 币种信息、质量评估、进场天数、各项指数
- **特点**: 主动发现市场机会，不限于用户收藏

**进场期质量变化通知**
- **触发条件**: 币种持续处于进场期，最新质量评估相对上一次机器人快照发生变化
- **通知内容**: 币种信息、质量变化方向、当前场外指数和爆破指数
- **特点**: 捕捉高质量、低质量、观察型进场之间的关键切换

**爆破指数由负转正通知**
- **触发条件**: 最新数据相对上一条数据从 `<=0` 转为 `>0`
- **通知内容**: 币种信息、爆破指数变化、当前状态
- **特点**: 捕捉市场情绪转暖信号

**强动能 / 短期撤出信号**
- **触发条件**: 原文动能符号出现 `$` 或 `‼`
- **通知内容**: 币种信息、信号含义、当前指标
- **特点**: 过滤普通高速油门和中等动能，降低打扰

### 个人收藏风险监控通知
**收藏币种爆破指数跌破200通知**
- **触发条件**: Dashboard收藏币种爆破指数从 `>=200` 跌到 `<200`，或原文出现 `‼`
- **监控来源**: 通过API实时获取用户的Dashboard收藏列表
- **通知内容**: 爆破指数数值、风险警告、当前状态

**收藏币种退场期提醒**
- **触发条件**: Dashboard收藏币种进入退场期第1天
- **监控来源**: 通过API实时获取用户的Dashboard收藏列表
- **通知内容**: 退场期天数、调仓建议、风险提示

## 数据库结构

机器人使用 SQLite 数据库存储用户数据，包含以下表：

- `users`: 用户基本信息和订阅状态
- `user_favorites`: 用户收藏的币种
- `notification_history`: 通知历史记录（防止重复通知）

## 定时任务

- **每天早上8:00**: 全市场机会发现
  - 高质量进场期机会轮询
  - 爆破指数由负转正提醒
- **每天晚上6:00**: 用户Dashboard收藏币种监控
  - 爆破指数跌破200警告
  - 退场期进入提醒

## 故障排除

### 常见问题

1. **机器人无响应**
   - 检查 Bot Token 是否正确
   - 确认网络连接正常
   - 查看控制台错误日志

2. **无法获取数据**
   - 确认 API_BASE_URL 配置正确
   - 检查主服务是否正常运行
   - 验证网络连接

3. **通知不发送**
   - 确认用户已订阅 (`/subscribe`)
   - 检查定时任务是否正常运行
   - 查看数据库中的通知历史记录

### 调试模式

## 扩展开发

### 添加新命令
在 `bot.js` 中添加新的命令处理器：
```javascript
bot.onText(/\/newcommand/, async (msg) => {
    // 处理逻辑
});
```

### 修改通知规则
编辑 `scheduler.js` 中的检查函数，调整通知条件和内容。

### 自定义定时任务
使用 node-cron 语法添加新的定时任务：
```javascript
cron.schedule('0 12 * * *', async () => {
    // 新的定时任务
}, {
    timezone: "Asia/Shanghai"
});
```
