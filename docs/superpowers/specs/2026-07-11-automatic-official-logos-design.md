# Automatic Official Logos Design

## Goal

Ensure newly added stock symbols automatically display an official company logo, including `GLW`, while preserving the existing crypto-logo behavior and a guaranteed local fallback.

## Architecture

The browser selects a logo source using the existing logo priority rules:

- Explicitly configured logo URLs retain highest priority.
- Existing synthetic indices, commodities, themes, and inline brand artwork continue through the backend `/api/logos/:symbol` route.
- Other symbols use Brandfetch identifier auto-detection, whose documented lookup order covers domains, stock and ETF tickers, ISINs, and crypto symbols.

Brandfetch images load directly in the browser because Brandfetch requires hotlinking and prohibits programmatic image downloading on the free Logo API. The Brandfetch Client ID is exposed as a runtime public configuration value; it is an identifier intended for client-side requests, not a secret API key.

## Data Flow

1. Coin data reaches the frontend with its symbol and optional explicit logo URL.
2. The logo resolver preserves explicit and existing special-asset sources.
3. Other symbols receive a Brandfetch auto-detection URL such as `https://cdn.brandfetch.io/GLW?c=<client-id>`.
4. Image loading failure switches to the deterministic SVG lettermark generated by the frontend.

New stock creation requires no symbol-to-domain mapping. `GLW` and future Yahoo Finance-backed stock symbols follow the same resolver path.

## Configuration

Add `BRANDFETCH_CLIENT_ID` as a server environment variable and expose it through the existing runtime app configuration as `BRANDFETCH_CLIENT_ID`. When the value is absent, the resolver uses the existing backend logo route, preserving current behavior.

Production deployment adds the Client ID to the Tokyo VPS container environment. No private Brandfetch secret is stored in frontend code.

## Error Handling

- Missing Client ID: use the existing backend logo route.
- Brandfetch image error or unresolved identifier: use the deterministic SVG lettermark.
- Existing explicit `data:` and `blob:` logo URLs retain priority.
- Existing manually configured logo URLs retain priority.

## Testing

- Unit test that `GLW` resolves to the Brandfetch auto-detection URL.
- Unit test that existing special assets continue to use the backend logo route.
- Unit test that missing Brandfetch configuration preserves the backend route.
- Component test that an image error switches to the SVG fallback where the current image component supports this behavior.
- Run the related logo tests, the full frontend test suite, and the production build.

## Scope

This change covers automatic logo selection and fallback. It excludes company metadata enrichment, bulk logo downloading, server-side Brandfetch caching, and asset-schema changes.
