- 添加 `time_precision` 字段存储时间精度级别
- 保持原有 `date` 字段的字符串格式
- 确保向后兼容性

## 支持的时间格式

### 输入格式
1. **日精度**：
   - `"5.9"` (5月9日，智能年份推断)
   - `"2024.5.9"` (2024年5月9日)
   - `"24.5.9"` (2024年5月9日，简化年份)

2. **小时精度**：
   - `"5.9 14"` (5月9日14时，智能年份推断)
   - `"2024.5.9 14"` (2024年5月9日14时)
   - `"24.5.9 14"` (2024年5月9日14时，简化年份)

3. **分钟精度**：
   - `"5.9 14:30"` (5月9日14时30分，智能年份推断)
   - `"2024.5.9 14:30"` (2024年5月9日14时30分)
   - `"24.5.9 14:30"` (2024年5月9日14时30分，简化年份)

4. **ISO格式**：`"2024-05-09T14:30:00"` (自动转换)

### 年份处理特性
- **智能年份推断**：当省略年份时，系统根据月份智能推断年份
  - 如当前1月，输入12月数据 → 推断为去年
  - 如当前12月，输入1月数据 → 推断为明年
  - 相近月份 → 推断为当年
- **完整年份支持**：2024.5.9
- **简化年份支持**：24.5.9 (自动转换为2024年)
- **跨年数据支持**：可以存储和查询不同年份的数据

### 数据库存储
- `date` 字段：保持原始格式字符串
- `timestamp` 字段：标准JavaScript Date对象
- `time_precision` 字段：枚举值 ('day', 'hour', 'minute')

## 使用示例

### 前端使用
```javascript

// 输入时间数据
const timeData = {
  date: '5.9 14:30',
  precision: 'minute'
};
```

### 后端处理
```javascript
// 解析时间
const timeInfo = parseFlexibleDateTime('5.9 14:30');
// 结果: { date: '5.9 14:30', timestamp: Date, precision: 'minute' }

// 存储到数据库
const metric = await DailyMetric.create({
  date: timeInfo.date,
  timestamp: timeInfo.timestamp,
  time_precision: timeInfo.precision,
  // ... 其他字段
});
```
